# Spiral 2 — System Design

## Step 1: Select the Architecture

We use **Layered Architecture**.

Presentation Layer

│

├── Console

├── API

└── UI

│

Business Layer

│

└── Services

│

Data Access Layer

│

└── Repository + DB Helper

│

Database

│

└── SQL Server

### Why?

- Separation of concerns
- Easy maintenance
- Easy testing
- Reusable code

- Layered Architecture

## Step 7: Error Handling Strategy

- Custom exceptions will be used where appropriate (e.g. ShipmentNotFoundException).
- API returns meaningful HTTP status codes (400, 404, 500).
- SQL exceptions are handled gracefully with try-catch blocks.
- Input is validated before database operations.
