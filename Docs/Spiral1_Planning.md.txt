## 2. Problem Statement

Delhivery's operations teams currently lack a unified internal tool to book, track, and analyse shipments across India. Without a centralized platform, shipment data is scattered, status updates are manual and error-prone, and daily analytics require time-consuming manual effort. The Delhivery Shipment Intelligence Platform (DSIP) aims to solve this by providing a single end-to-end system covering shipment booking, real-time tracking, status management, and automated end-of-day reporting.

---

## 3. Project Objectives

1. Book new shipments with validated data (AWB, sender, receiver, origin, destination, weight)
2. Track shipments by AWB number in real time
3. Update shipment status through a defined lifecycle (Booked → In Transit → Out for Delivery → Delivered / RTO)
4. Cancel shipments when required
5. Provide a dashboard with live statistics (counts per status)
6. Filter and view shipments by status
7. Generate automated end-of-day analytics reports with CSV export
8. Demonstrate responsible use of AI tools with proper logging and reflection

---

## 4. Project Scope

### In Scope

- C# Console Application (in-memory shipment prototype)
- SQL Server Database (schema, stored procedures, views, indexes, seed data)
- ADO.NET Data Access Layer (parameterized queries, no ORM)
- ASP.NET Core Web API (6 RESTful endpoints with validation)
- Browser-based UI Dashboard (HTML/CSS/JS/jQuery, single page, AJAX-driven)
- Python Analytics Script (end-of-day report with CSV export)
- Gen AI Documentation (Copilot log, prompt engineering, reflection)

### Out of Scope

- User authentication / login system
- Payment gateway integration
- Real-time GPS tracking
- Mobile application
- Email / SMS notifications
- Multi-tenant architecture
- Production deployment / CI-CD pipeline
